# Описание программного комплекса для магистерской диссертации

**Тема:** интеллектуальная система автоматического распознавания объектов на спутниковых изображениях с использованием глубокого обучения и архитектуры YOLO.

**Версия программы:** 1.0.0  
**Язык реализации:** Python 3.11+  
**Основные библиотеки:** PyTorch, Ultralytics YOLOv8, OpenCV, NumPy, Pandas.

Документ предназначен для включения в пояснительную записку (разделы «Описание программы», «Алгоритм работы», «Метрики оценки»).

---

## 1. Назначение программы

Разработанный программный комплекс предназначен для:

- автоматической обработки спутниковых изображений форматов JPG, PNG, TIFF;
- обнаружения и локализации объектов с помощью ограничивающих рамок (bounding box);
- классификации объектов по заданным категориям;
- визуализации и сохранения результатов распознавания;
- обучения и дообучения нейросетевой модели на размеченных данных;
- расчёта метрик качества распознавания.

Область применения — анализ данных дистанционного зондирования Земли (аэрофотосъёмка, снимки со спутников).

---

## 2. Структура программной системы

Программа организована в виде модулей и исполняемых скриптов:

| Компонент | Назначение |
|-----------|------------|
| `preprocess/` | Предобработка и аугментация изображений |
| `train/` | Обучение модели YOLO (transfer learning, сохранение весов) |
| `inference/` | Инференс: детекция, NMS, экспорт результатов |
| `evaluate/` | Расчёт метрик Precision, Recall, mAP и др. |
| `api/`, `web/` | REST API и веб-интерфейс |
| `train.py` | Запуск обучения |
| `detect.py` | Запуск распознавания на изображениях |
| `evaluate.py` | Оценка качества обученной модели |
| `preprocess.py` | Пакетная предобработка |
| `configs/default.yaml` | Параметры системы |
| `dataset.yaml` | Описание датасета и классов |

Данные для обучения хранятся в каталоге `dataset/` в формате YOLO (пары «изображение — файл разметки»). Обучение выполнялось на подмножестве датасета **DOTA** (train: 777 снимков, val: 409 снимков после конвертации и фильтрации классов).

## 3. Архитектура нейросетевой модели

В основе системы — одноэтапный детектор **YOLOv8** (You Only Look Once, версия 8), реализованный в библиотеке Ultralytics. Архитектура включает три логических блока:

1. **Backbone (реверсивная свёрточная сеть)** — извлечение признаков из входного изображения; формирование карт признаков (feature maps) низкого и высокого уровней.
2. **Neck (FPN / PAN)** — объединение признаков разных масштабов для устойчивого обнаружения объектов разного размера.
3. **Detection Head** — предсказание для каждой ячейки сетки:
   - координат ограничивающей рамки (bounding box);
   - оценки уверенности (confidence);
   - вероятностей принадлежности к классам.

По умолчанию используется вариант **YOLOv8m** (medium) с предобученными весами; для отладки и работы на CPU допускается **YOLOv8n** (nano). Применяется **transfer learning**: начальные веса загружаются с задачи общей детекции (COCO), затем модель дообучается на спутниковых данных DOTA.

## 4. Этапы алгоритма распознавания

Алгоритм работы системы реализован как последовательный **pipeline** из шести этапов.

### Этап 1. Получение входного изображения

- Загрузка файла с диска или через API.
- Поддерживаемые форматы: JPG, JPEG, PNG, TIF, TIFF.
- Проверка размера файла, формата и разрешения (модуль `utils/security.py`).

### Этап 2. Предварительная обработка

Выполняется модулем `ImagePreprocessor` (`preprocess/pipeline.py`):

| Операция | Описание | Параметры |
|----------|----------|-----------|
| Подавление шума | Фильтр Non-local Means | `denoise: true` |
| Коррекция яркости и контраста | Линейное преобразование пикселей | α = 1.1, β = 5 |
| Изменение размера | Масштабирование с сохранением пропорций и паддинг до квадрата | 640×640 px |
| Нормализация | Приведение к диапазону [0, 1], подготовка тензора | RGB, порядок CHW |

Для изображений, превышающих порог по размеру (больше 2048 px по большей стороне), применяется **разбиение на тайлы** (tiling): окно 1024×1024 px с перекрытием 128 px.

### Этап 3. Извлечение признаков (Backbone)

Входной тензор передаётся в свёрточную сеть YOLOv8. Автоматически формируются многомасштабные карты признаков; явное программирование признаков не требуется — используется представление, обученное end-to-end.

**Используемые характеристики на этом этапе** (неявно, внутри сети):

- низкоуровневые: границы, текстуры, контрастные области;
- высокоуровневые: формы объектов, контекст местности (дороги, водоёмы, застройка).

### Этап 4. Обнаружение объектов (Detection Head)

Detection Head выдаёт для каждого кандидата:

- координаты bounding box \((x_1, y_1, x_2, y_2)\);
- **confidence score** — уверенность в наличии объекта;
- **class ID** — индекс класса из семи категорий.

Порог отсечения по уверенности: **conf_threshold = 0.25**.

### Этап 5. Постобработка

- **Non-Maximum Suppression (NMS)** — удаление дублирующихся рамок с пересечением выше порога **IoU threshold = 0.45**;
- фильтрация по confidence;
- при использовании tiling — дополнительное объединение детекций между фрагментами изображения.

### Этап 6. Формирование результата

- отрисовка рамок, подписей класса и confidence на изображении;
- экспорт в форматы: **JPG** (визуализация), **JSON**, **CSV**, **TXT**.

## 6. Обучение модели

### 6.1. Данные

- Источник: датасет **DOTA** (ориентированные bounding box конвертируются в горизонтальные YOLO-формат).
- Разбиение: **train** (обучение), **val** (валидация и метрики).
- Формат разметки YOLO:  
  `class_id x_center y_center width height`  
  координаты нормализованы в интервале [0, 1].

### 6.2. Эпохи обучения

**Эпоха (epoch)** — один полный проход алгоритма оптимизации по всем обучающим изображениям.

| Режим | Число эпох | Назначение |
|-------|------------|------------|
| Отладка (`configs/debug.yaml`) | 3 | Проверка работоспособности pipeline |
| Основной (`configs/default.yaml`) | **100** | Полное обучение модели |
| Early stopping | patience = **20** | Остановка, если метрика на val не улучшается 20 эпох подряд |

Дополнительно:

- **warmup_epochs = 3** — плавный разогрев скорости обучения в начале;
- **save_period = 10** — сохранение промежуточных checkpoint каждые 10 эпох;
- лучшие веса сохраняются в `models/best.pt` по метрике на валидации.

### 6.3. Гиперпараметры обучения

| Параметр | Значение | Назначение |
|----------|----------|------------|
| Оптимизатор | AdamW | Адаптивное обновление весов |
| Начальная скорость обучения lr0 | 0.001 | Шаг градиентного спуска |
| Конечная доля lr (lrf) | 0.01 | Снижение lr к концу обучения |
| Размер батча | 16 (4 на CPU) | Число изображений за одну итерацию |
| Размер входа | 640×640 | Согласован с предобработкой |
| Transfer learning | включён | Инициализация предобученными весами |
| AMP | включён (GPU) | Ускорение смешанной точностью |

### 6.4. Аугментация данных (только при обучении)

Повышает устойчивость модели к вариациям съёмки:

| Метод | Параметр |
|-------|----------|
| Поворот | ±15° |
| Отражение по горизонтали | 50% |
| Масштабирование | scale 0.5 |
| Изменение оттенка/насыщенности/яркости (HSV) | h, s, v |
| Mosaic | 1.0 |
| MixUp | 0.1 |

## 7. Метрики оценки качества

Метрики рассчитываются модулем `evaluate/metrics.py` на **валидационной выборке** с использованием эталонной разметки. Сравниваются предсказанные и истинные bounding box.

### 7.1. Основные метрики

| Метрика | Обозначение | Смысл |
|---------|-------------|--------|
| **Precision (точность)** | P | Доля верных срабатываний среди всех детекций: P = TP / (TP + FP) |
| **Recall (полнота)** | R | Доля найденных объектов среди всех объектов на снимке: R = TP / (TP + FN) |
| **F1-score** | F1 | Гармоническое среднее: F1 = 2PR / (P + R) |
| **IoU** | — | Intersection over Union — степень пересечения предсказанной и эталонной рамки |
| **mAP50** | mAP@0.5 | Средняя точность (Average Precision) при пороге IoU = 0.50, усреднённая по классам |
| **mAP50-95** | mAP@0.5:0.95 | Средняя AP при IoU от 0.50 до 0.95 с шагом 0.05 (метрика COCO) |

**TP** — истинно положительное обнаружение (IoU ≥ порога и верный класс);  
**FP** — ложное срабатывание;  
**FN** — пропущенный объект.

### 7.2. Требования к качеству (по ТЗ)

| Метрика | Минимальное значение |
|---------|----------------------|
| mAP50 | ≥ 0.75 |
| Precision | ≥ 0.80 |
| Recall | ≥ 0.75 |

Программа автоматически формирует отчёт `results/evaluation_report.json` с флагом выполнения требований.

### 7.3. Когда считаются метрики

- после завершения обучения (`train.py`);
- отдельной командой `evaluate.py --weights models/best.pt`;
- через REST API (`GET /metrics`) — последний сохранённый отчёт.

**Важно:** метрики на классах DOTA осмыслены только после **дообучения** модели на размеченных train/val. Предобученная модель COCO на спутниковых снимках даёт низкие значения mAP.

## 8. Характеристики входных и выходных данных

### 8.1. Вход

- спутниковые RGB-изображения;
- разрешение исходных снимков DOTA — до нескольких тысяч пикселей по стороне;
- при инференсе — один файл или пакет файлов.

### 8.2. Выход (на каждый снимок)

| Поле | Описание |
|------|----------|
| `bbox` | Координаты рамки [x1, y1, x2, y2] в пикселях |
| `class_name` | Название класса |
| `class_id` | Числовой идентификатор 0–6 |
| `confidence` | Уверенность модели, 0–1 |
| `num_detections` | Число обнаруженных объектов |

---

## 9. Режимы работы программы

1. **Обучение** — `python train.py --data dataset.yaml --epochs 100`
2. **Распознавание** — `python detect.py --source <путь к изображению> --weights models/best.pt`
3. **Оценка метрик** — `python evaluate.py --weights models/best.pt`
4. **Веб-сервис** — `python run_api.py` (интерфейс и REST API)

---

## 10. Программная реализация (кратко для записки)

Разработанная система реализует полный цикл: подготовка данных DOTA → обучение YOLOv8 с аугментацией и transfer learning → инференс с предобработкой и NMS → расчёт метрик Precision, Recall, F1, mAP50, mAP50-95. Архитектура соответствует современным одноэтапным детекторам объектов и применима к задачам мониторинга объектов на спутниковых снимках.

## 11. Схема алгоритма (для иллюстрации в диссертации)

```text
[Входное изображение]
        ↓
[Загрузка и проверка формата]
        ↓
[Предобработка: шум, яркость, resize 640]
        ↓
[Tiling — при большом разрешении]
        ↓
[YOLOv8: Backbone → Neck → Detection Head]
        ↓
[Фильтр confidence ≥ 0.25]
        ↓
[Non-Maximum Suppression, IoU ≥ 0.45]
        ↓
[Визуализация + JSON / CSV / TXT]
```
